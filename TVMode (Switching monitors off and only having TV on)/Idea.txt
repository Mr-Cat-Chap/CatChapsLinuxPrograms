The most common reason this fails—even if the echo looks correct—is that hardcoding a line number (NR == 52) is unreliable and awk may be capturing hidden whitespace or carriage returns that break the exact string comparison.[1][2][3]

Here are the three main reasons it isn't working and how to fix them:

1. The "Hidden Character" Problem

If kscreen-doctor output contains a carriage return (\r) or trailing spaces, TVStatus will be "disabled\r" instead of "disabled". While echo might show "disabled", the if statement will fail because they aren't an exact match.

The Fix: Use [[ ]] (which is more flexible) and trim whitespace, or use awk's index or a regex match.

code
Bash

content_copy
expand_less
# Trim any potential hidden characters or whitespace
TVStatus=$(kscreen-doctor -o | awk 'NR == 52 {gsub(/[[:space:]]/,""); print $1}')

# Use double brackets for safer string comparison
if [[ "$TVStatus" == "disabled" ]]; then
    echo "Yes"
fi
2. Hardcoding NR == 52 is Fragile

Using line 52 is very risky.[1][2][3][5][6] If you plug in a different monitor or your GPU driver lists one extra resolution mode, "line 52" will shift to a different piece of data.[2][3][5] Your script might be accidentally grabbing a word like "HDR:" or a resolution instead of the status.[1][5][6]

The Fix: Search for the specific monitor name and then look for the word "enabled" or "disabled" on that same line.[3][6]

3. Missing Quotes in [ ]

In standard Bash [ ] tests, if $TVStatus is empty or contains a space, the script will crash or return an error.[2][5] You should always wrap variables in double quotes.[2][3][5][6]

The Robust Way (Recommended)

Instead of guessing which line the TV is on, tell awk to find the line for your specific TV (e.g., HDMI-A-1) and check its status directly.[2][5]

code
Bash

content_copy
expand_less
# 1. Find the status of a specific output (e.g., HDMI-A-1)
# This looks for the line containing your TV name and checks if "disabled" is present.
TVStatus=$(kscreen-doctor -o | awk '/HDMI-A-1/ {print ($0 ~ /disabled/ ? "disabled" : "enabled")}')

echo "TV is $TVStatus"

# 2. Compare safely
if [[ "$TVStatus" == "disabled" ]]; then
    echo "Yes, it is disabled."
fi
